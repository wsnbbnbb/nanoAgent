## 一、原项目分析

### 1.1 NanoAgent 整体架构

NanoAgent 是一个**轻量级模块化 Agent 框架**，其架构设计遵循**关注点分离（Separation of Concerns）**原则，将核心功能划分为多个独立模块：

**核心模块划分：**

| 模块 | 路径 | 职责 |
|------|------|------|
| **Core** | `agent/core/` | Agent 核心运行逻辑、任务执行器 |
| **Tools** | `agent/tools/` | 工具定义、文件系统操作、搜索、命令执行、规划 |
| **Context** | `agent/context/` | Token 计数、上下文压缩、会话管理 |
| **Memory** | `agent/memory/` | 长期记忆管理（向量存储） |
| **Loaders** | `agent/loaders/` | 外部配置加载（规则、技能、MCP） |
| **Teams** | `agent/teams.py` | 多 Agent 协作管理 |
| **MCP** | `agent/mcp/` | Model Context Protocol 支持 |

**架构层次图（文字描述）：**

```
┌─────────────────────────────────────────────────────────┐
│                     AgentRunner                          │
│              (主入口：run/run_step)                      │
└─────────────────────┬───────────────────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
   ┌─────────┐   ┌─────────┐   ┌──────────┐
   │ Loader  │   │ Executor│   │ LLM API  │
   │ 模块    │   │   模块   │   │  (OpenAI)│
   └────┬────┘   └────┬────┘   └──────────┘
        │            │
   ┌────┴────┐   ┌────┴────┐
   │Rules    │   │Tools    │
   │Skills   │   │Registry │
   │MCP      │   └────┬────┘
   └─────────┘        │
                  ┌────┴────┐
                  │Tool Impl│
                  │(Base类) │
                  └─────────┘
```

### 1.2 Agent 核心工作机制

NanoAgent 的核心执行流程遵循 **ReAct (Reasoning + Acting)** 范式的简化版本：

**执行流程（数据流）：**

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ 1. 上下文加载（_load_context）                           │
│    - 加载 Rules → 系统提示词增强                          │
│    - 加载 Skills → 工具能力描述                          │
│    - 加载 Memory → 历史会话摘要                         │
│    - 加载 MCP Tools → 扩展工具集                         │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│ 2. 消息构建（build_system_prompt）                        │
│    - 组装 system message（含 rules/skills/memory）       │
│    - 添加用户输入作为 user message                       │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│ 3. ReAct 循环执行（run_step）                            │
│    for iteration in range(max_iterations):               │
│        │                                                │
│        ├── LLM.chat.completions.create()               │
│        │    (发送消息列表 → 获取工具调用/文本响应)        │
│        │                                                │
│        ├── 无工具调用？→ 返回文本结果                    │
│        │                                                │
│        └── 有工具调用？→ 遍历执行                        │
│                      │                                   │
│                      ├── 解析函数名 + JSON 参数          │
│                      ├── 执行 Tool（函数映射表查找）      │
│                      ├── 捕获返回值                       │
│                      └── 添加入 messages（role=tool）    │
│                                                          │
│    循环直到：无工具调用 或 达到最大迭代次数                │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
                 ┌───────────────┐
                 │ 保存执行结果   │
                 │ 到 Memory     │
                 └───────────────┘
```

**关键设计特点：**

1. **函数映射表模式**：`available_functions` 字典维护 `tool_name → function_impl` 的映射，执行时直接查找调用
2. **迭代式执行**：通过 `max_iterations` 限制单次任务的最大工具调用次数
3. **计划模式（Plan）**：支持将复杂任务分解为多个子步骤，递归调用 `run_step` 执行
4. **上下文累积**：每次工具调用的结果都追加到 messages 列表，形成完整的执行历史

### 1.3 工具系统设计

NanoAgent 采用**基类继承 + JSON 协议**的工具设计：

```python
class Tool(ABC):
    name: str
    description: str
    parameters: List[ToolParameter]
    
    @abstractmethod
    def run(self, parameters: Dict[str, Any]) -> str:
        """执行工具，返回 JSON 格式的标准化响应"""
        pass
```

**标准化响应协议：**

```json
{
  "status": "success|error|partial",
  "error_code": "INVALID_PARAM|NOT_FOUND|...",
  "content": "实际返回内容",
  "metadata": { /* 附加信息 */ }
}
```

**基础工具集：**

| 工具 | 功能 | 安全性 |
|------|------|--------|
| `read` | 读取文件（支持偏移/限制） | 仅文件存在性校验 |
| `write` | 覆盖写入文件 | 无校验 |
| `edit` | 字符串替换 | old_string 唯一性校验 |
| `glob` | 文件模式匹配 | 无校验 |
| `grep` | 正则搜索 | 直接调用 grep 命令 |
| `bash` | Shell 执行 | 无黑名单限制 |
| `plan` | 任务拆解 | 调用 LLM |

### 1.4 上下文管理机制

NanoAgent 的 `ContextManager` 实现了**基于 Token 预算的自动压缩**：

```python
class ContextManager:
    DEFAULT_CONTEXT_WINDOW = 128000  # Token 上限
    COMPRESSION_THRESHOLD = 0.8      # 压缩触发阈值（80%）
    MIN_RETAIN_MESSAGES = 6          # 保留最小消息数
```

**压缩策略：**
- 当预估 token 数达到上限的 80% 时触发压缩
- 保留最近 N 条消息 + 生成历史摘要
- 摘要可通过 LLM 生成或使用简单规则（消息计数统计）

#### 2.2.1 ContextBuilder：分层消息构建

MyCodeAgent 的 `ContextBuilder` 实现了**四层消息架构**：

```python
# L1: System prompt + Tools（全局不变，缓存）
# L2: CODE_LAW 项目规则（可变，mtime 缓存）
# L3: History messages（会话历史，动态累积）
```

### 3.3 关键功能实现思路

#### 3.3.1 工具调用增强

**原实现（函数映射表）：**

```python
# agent-claude-code.py
available_functions = {
    "read": read,
    "write": write,
    "bash": bash,
    # ...
}

# 执行时直接查找
function_impl = available_functions.get(function_name)
if function_impl:
    function_response = function_impl(**function_args)
```

**拓展实现（ToolRegistry）：**

```python
# MyCodeAgent-lite
class ToolRegistry:
    def __init__(self):
        self._tools: Dict[str, Tool] = {}
        self._circuit_breaker = CircuitBreaker()
    
    def register_tool(self, tool: Tool):
        self._tools[tool.name] = tool
    
    def execute(self, tool_name: str, parameters: Dict) -> str:
        # 熔断检查
        if self._circuit_breaker.is_open(tool_name):
            return f"Tool '{tool_name}' is temporarily disabled"
        
        tool = self._tools.get(tool_name)
        if not tool:
            return f"Unknown tool: {tool_name}"
        
        try:
            result = tool.run(parameters)
            self._circuit_breaker.record_success(tool_name)
            return result
        except Exception as e:
            self._circuit_breaker.record_failure(tool_name)
            return f"Error: {str(e)}"
```

**设计优势：**
- **可扩展性**：新增工具只需实现 Tool 基类并注册
- **可观测性**：熔断状态可监控
- **一致性**：统一的错误处理和响应格式

#### 3.3.2 Bash 安全沙箱实现

```python
class SecureBashTool(Tool):
    INTERACTIVE_COMMANDS = {"vim", "nano", "ssh", ...}
    DESTRUCTIVE_COMMANDS = {"mkfs", "dd", "shutdown", ...}
    
    def run(self, parameters: Dict) -> str:
        command = parameters.get("command", "")
        
        # 第一层：命令黑名单
        first_word = command.strip().split()[0]
        if first_word in self.INTERACTIVE_COMMANDS:
            return self._error("Interactive commands not allowed")
        
        # 第二层：目录沙箱
        resolved_dir = self._resolve_and_validate_dir(parameters.get("directory"))
        
        # 第三层：超时控制
        timeout_ms = min(parameters.get("timeout_ms", 120000), 600000)
        
        # 执行
        result = subprocess.run(
            command,
            shell=True,
            capture_output=True,
            timeout=timeout_ms/1000,
            cwd=resolved_dir
        )
        
        return self._format_output(result)
```

```python
class PlanningTool(Tool):
    def run(self, parameters: Dict) -> str:
        task = parameters.get("task")
        
        # 1. LLM 生成计划
        plan_data = self._llm_generate_plan(task)
        steps = plan_data.get("steps", [])
        
        # 2. 返回计划待审批（不自动执行）
        return json.dumps({
            "status": "pending_approval",
            "steps": steps,
            "estimated_steps": len(steps)
        })
    
    def approve_plan(self, plan_id: str):
        """审批通过后执行"""
        steps = self._get_plan_steps(plan_id)
        results = []
        for step in steps:
            result = self._execute_step(step)
            results.append(result)
        return results
```

### 3.6 核心逻辑伪代码

#### Agent 主循环

```python
class AgentRunner:
    def run(self, task: str, use_plan: bool = False) -> str:
        # 1. 初始化
        rules, skills, memory = self._load_context()
        system_prompt = self._build_system_prompt(rules, skills, memory)
        messages = [{"role": "system", "content": system_prompt}]
        messages.append({"role": "user", "content": task})
        
        # 2. 计划模式（可选）
        if use_plan:
            return self._run_with_planning(task, messages)
        
        # 3. 标准 ReAct 循环
        return self._run_react_loop(messages)
    
    def _run_react_loop(self, messages: List[Dict]) -> str:
        for iteration in range(self.max_iterations):
            # LLM 调用
            response = self.llm.chat.completions.create(
                model=self.model,
                messages=messages,
                tools=self.tool_registry.get_openai_tools()
            )
            
            assistant_msg = response.choices[0].message
            messages.append(assistant_msg)
            
            # 无工具调用 → 结束
            if not assistant_msg.tool_calls:
                return assistant_msg.content
            
            # 工具调用循环
            for tool_call in assistant_msg.tool_calls:
                result = self.tool_registry.execute(
                    tool_call.function.name,
                    tool_call.function.arguments
                )
                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": result
                })
        
        return "Max iterations reached"
```

#### 工具注册与执行

```python
class ToolRegistry:
    def __init__(self):
        self._tools: Dict[str, Tool] = {}
        self._circuit_breaker = CircuitBreaker()
    
    def register(self, tool: Tool):
        """注册工具"""
        self._tools[tool.name] = tool
    
    def execute(self, name: str, params: Dict) -> str:
        """执行工具（带保护）"""
        # 熔断检查
        if self._circuit_breaker.is_open(name):
            return json.dumps({
                "status": "error",
                "error_code": "CIRCUIT_OPEN",
                "content": f"Tool '{name}' temporarily disabled"
            })
        
        tool = self._tools.get(name)
        if not tool:
            return json.dumps({
                "status": "error",
                "error_code": "NOT_FOUND",
                "content": f"Unknown tool: {name}"
            })
        
        try:
            result = tool.run(params)
            self._circuit_breaker.record_success(name)
            return result
        except Exception as e:
            self._circuit_breaker.record_failure(name)
            return json.dumps({
                "status": "error",
                "error_code": "INTERNAL_ERROR",
                "content": str(e)
            })
    
    def get_openai_tools(self) -> List[Dict]:
        """生成 OpenAI Function Calling 格式"""
        return [
            {"type": "function", "function": tool.get_definition()}
            for tool in self._tools.values()
        ]
```

## 四、总结与不足

### 4.1 当前实现存在的问题

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| **Bash 安全性不足** | 高 | agent-claude-code 的 bash 工具无任何保护 |
| **上下文管理简陋** | 中 | 仅保留最后 50 行，无智能压缩 |
| **错误处理粗糙** | 中 | 大部分异常直接返回字符串 error |
| **无会话持久化** | 中 | 每次运行从头开始 |
| **工具发现性差** | 低 | 依赖字典查找，无注册表 |
| **无熔断机制** | 中 | 故障工具会持续失败 |

### 4.2 与完整 MyCodeAgent 的差距

| 功能 | MyCodeAgent | MyCodeAgent-lite | 差距 |
|------|-------------|------------------|------|
| **ReAct 执行** | ✅ 完整循环 | ⚠️ 简化版 Plan | 中 |
| **Bash 安全** | ✅ 多层防护 | ❌ 无防护 | 大 |
| **上下文压缩** | ✅ HistoryManager | ⚠️ ContextManager | 中 |
| **熔断器** | ✅ CircuitBreaker | ❌ 无 | 大 |
| **Agent Teams** | ✅ TeamManager | ⚠️ 简化 AgentTeam | 中 |
| **Skill 系统** | ✅ SkillLoader | ⚠️ 静态 JSON | 中 |
| **Trace 日志** | ✅ TraceLogger | ❌ 无 | 大 |
| **MCP 支持** | ✅ 完整 loader | ⚠️ 基础加载 | 小 |
